Mirus — Gemelo digital turístico de El Salvador

Plataforma web que unifica mapa interactivo 3D, catálogo oficial MITUR, rutas inteligentes y experiencia personalizada para planificar y explorar destinos nacionales — con caso piloto en Barra de Santiago y cobertura de 104 lugares en 13 departamentos.

Producto: Mirus Repositorio: TwinMap GitHub: RebecaCAstro22/twinmap Demo: mirus-blue.vercel.app
● Demo en producción 104 destinos MITUR Mapbox GL 3D Low twin · sin IoT

Visión

Mirus es un gemelo digital turístico de enfoque low twin: combina datos abiertos, cachés geoespaciales y simulación con reglas realistas para ofrecer una experiencia cartográfica rica sin depender de sensores IoT ni modelos de ML entrenados. El mapa es el núcleo obligatorio; encima se superponen capas de diferenciación — afluencia, biodiversidad, tráfico, noticias y rutas personalizadas — que convierten la información fragmentada del turismo salvadoreño en una sola interfaz usable por viajeros, comunidades locales y evaluadores del reto TWINMAP.

Propuesta de valor: Convierte las preferencias del viajero en mapa vivo, itinerario editable y ruta lista para recorrer El Salvador, con datos reales cuando el backend está activo y continuidad offline mediante almacenamiento local por perfil.

Problema y solución

El problema

La información para viajar por El Salvador está dispersa entre mapas genéricos, redes sociales, guías desactualizadas y fuentes institucionales poco integradas. El viajero no dispone de una vista unificada que combine destinos oficiales, contexto ambiental (clima, biodiversidad), afluencia estimada, movilidad local y alertas de seguridad en tiempo casi real.

La solución

Mirus integra dos piezas complementarias en un mismo repositorio: una aplicación Mirus (landing, login, cuestionario, bitácora, favoritos, emergencias) y un gemelo cartográfico Mapbox con más de una docena de capas activables. La app embebe el mapa vía iframe y sincroniza favoritos y visitas mediante postMessage y almacenamiento compartido por perfil.

Funcionalidades implementadas

Stack tecnológico

  • Frontend: HTML5, CSS3, JavaScript vanilla modular (20+ módulos en frontend/src/js/); sin framework ni build step.
  • Mapas: Mapbox GL JS v3.17, estilo satellite-streets-v12, geocoder v5, capas vectoriales de tráfico, edificios 3D con fill-extrusion.
  • Backend: Node.js, Express 5, ES modules, axios, dotenv.
  • Scripts de datos: Node (.mjs) y Python (pytrends) para regenerar cachés.
  • Persistencia: localStorage por perfil (TwinmapPlaceStorage), GeoJSON/JSON estáticos servidos por Express.
  • Integración app ↔ mapa: iframe + postMessage para capas, favoritos y bitácora.

Caso piloto y cobertura nacional

El gemelo digital se diseñó con foco en Barra de Santiago (humedal RAMSAR, manglar, ecoturismo comunitario, aves migratorias y tortugas) — centro del mapa en lat 13.74, lng -90.05. La capa de biodiversidad y naturaleza curada reflejan este ecosistema costero.

Simultáneamente, la integración del catálogo MITUR eleva la plataforma a escala nacional: 104 destinos oficiales distribuidos en 13 departamentos y 26 categorías (playas, volcanes, cascadas, sitios arqueológicos, gastronomía, etc.), con heatmap de afluencia ponderado por pesos turísticos institucionales.

Despliegue

Frontend — Vercel

Sitio estático desplegado en mirus-blue.vercel.app. Build con scripts/netlify-config.mjs que genera config.js desde variables MAPBOX_TOKEN y API_BASE. Rewrites: / → app Mirus, /map → gemelo standalone.

Backend — Railway / Render

API Express en puerto configurable (PORT). Railway puede servir frontend + API desde el mismo proceso. Alternativas: Render, VPS propio. Variables en .env (Mapbox, SerpAPI, eBird, OpenRouter, NewsData, n8n webhook).

Desarrollo local: npx serve -l 3000 . (estático) + npm run dev (API :3001). La UI degrada gracefully si el backend no está disponible.

  • Catálogo MITUR completo (104 lugares) con filtros por categoría, heatmap mejorado y símbolos diferenciados.

  • Movilidad local: enlaces directos a Waze e inDrive desde el sidebar de cada lugar.
  • Noticias y emergencias con NewsData.io en vivo, fallback a Google Noticias y contactos oficiales de El Salvador.
  • Login Google además de perfiles locales; bitácora y favoritos editables.
  • Backend modular con 7 routers, cachés GeoJSON versionados y degradación a mocks.
  • Demo pública desplegada en Vercel, lista para evaluación de jurado.
  • Enfoque low twin creíble: datos abiertos + simulación explicable, sin prometer IoT inexistente.
